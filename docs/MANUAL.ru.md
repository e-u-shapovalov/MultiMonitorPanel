# MultiMonitorPanel — мануал

*(English version: [MANUAL.en.md](MANUAL.en.md))*

---

## Что это

**MultiMonitorPanel** — portable-панель запуска для Windows 10/11: свой док на
**каждый** монитор, и у каждого дока свой набор кнопок. В отличие от таскбара
Windows, разные мониторы не обязаны показывать одинаковые закреплённые иконки.
Это настоящая системная панель (**AppBar**): развёрнутые окна её не перекрывают —
Windows вырезает под неё полосу из рабочей области экрана.

Один нативный `.exe` меньше 400 КБ без рантайм-зависимостей (GDI+, comctl32, shcore
есть в любой Windows 10/11).

## Скачать релиз

1. Открой страницу
   [Releases / Latest](https://github.com/e-u-shapovalov/MultiMonitorPanel/releases/latest).
2. В блоке **Assets** скачай готовый файл программы. В текущем релизе
   это `panel.exe`.
3. Положи `panel.exe` в постоянную папку, например
   `C:\Tools\MultiMonitorPanel`.
4. Запусти `panel.exe`. Папка `ico` создастся рядом с ним при первом запуске;
   туда можно класть свои иконки.

Если в будущем релиз будет выложен как архив `MultiMonitorPanel-*-win64.zip`,
скачай именно архив из **Assets** и распакуй его перед запуском.

Если ты обычный пользователь, не нажимай **Code → Download ZIP** и не скачивай
**Source code**. Это исходный код для разработчиков, а не готовая программа.

Если Windows SmartScreen предупреждает о неизвестном приложении, проверь, что
файл скачан именно из этого репозитория, затем нажми **Подробнее → Выполнить в
любом случае**.

## Зачем создано

Когда мониторов два или три, одинаковые закреплённые иконки на каждом таскбаре
мешают: на одном экране нужны папки и задачи, на другом IDE, на третьем терминалы,
админские утилиты или удалённые подключения. MultiMonitorPanel создавался как
простая нативная альтернатива тяжёлым комбайнам: отдельная панель на каждый
экран, без .NET, Electron, скинов и постоянного фонового мусора.

## Как работает

При старте программа определяет мониторы слева направо, создаёт на каждом
системную AppBar-панель и читает настройки из
`HKCU\Software\MultiMonitorPanel`. При клике кнопка запускает цель через Windows
Shell, а новое окно переносится в рабочую область того монитора, где была нажата
кнопка. Для `cmd`, PowerShell и других консолей есть флаг **Консоль**: он
заставляет использовать классический `conhost`, чтобы окно не уехало на монитор,
выбранный Windows Terminal.

## Запуск

- Двойной клик по `panel.exe`.
- Второй запуск молча выходит (работает один экземпляр).
- **Автозапуск:** положи ярлык `panel.exe` в папку автозагрузки
  (`Win+R` → `shell:startup`).
- При подключении/отключении монитора панели сами пересоздаются.

## Меню (правый клик по панели)

| Пункт | Что делает |
|-------|-----------|
| **Изменить настройки…** | открыть окно-редактор настроек (см. ниже) |
| **Перезагрузить настройки** | перечитать настройки и перерисовать панели |
| **Образцовый набор** | записать стандартный образцовый набор (на 3 монитора) — в нём показаны все возможности |
| **Стереть настройки** | убрать все кнопки (панели станут пустыми) |
| **Язык / Language** | переключить язык интерфейса |
| **Открыть папку программы (ico)** | открыть папку с `panel.exe` (там же папка `ico`) |
| **О программе** | версия, дата релиза, автор, ссылка на репозиторий |
| **Выход** | закрыть программу |

## Редактор настроек

ПКМ → **Изменить настройки…**. Окно правит настройки на месте — без `regedit` и
текстовых файлов. Окно **растягивается** мышью за край (длинные аргументы видно
целиком).

- **Сверху:** выбор монитора (кнопки **+/−** добавляют и убирают монитор — можно
  настроить экран, который сейчас не подключён; кнопка **⇄** меняет набор кнопок
  текущего монитора местами с другим), высота полосы, папка иконок.
- **Слева:** список кнопок выбранного монитора, сгруппированный по краю —
  секции «слева», «центр», «справа».
- **Справа:** поля выбранной кнопки — подпись, что запускать, аргументы, иконка
  (кнопка `…` — выбрать файл), **край** (к какому краю полосы прижать кнопку) и
  галочки **админ** / **консоль** / **разделитель**.
- **Добавить** / **+ Разделитель** / **Удалить** / **▲** / **▼** — менять состав
  и порядок. **▲/▼** на границе секции переносят запись в соседнюю группу.
- **Из панели задач…** (внизу слева) — забрать закреплённые в панели задач Windows
  программы кнопками на текущий монитор: отметь нужные в диалоге (Ctrl-клик —
  сразу несколько). В самой панели задач Windows они остаются.
- **ПКМ по кнопке в списке** — **Копировать** / **Вставить**, либо **Переместить
  на монитор** — сразу перенести кнопку на другой экран, без копирования.
- Разделитель берёт край от того места, где стоит — двигай его **▲/▼** в нужную
  секцию (его переключатель края неактивен).
- **OK** или **Применить** — сразу записать и перерисовать панели (Применить не
  закрывает окно). **Отмена** — выйти без записи.

## Где живут настройки

В реестре: `HKCU\Software\MultiMonitorPanel` (не в файле). Правятся через
окно-редактор. Источник истины — реестр; редактор по OK/Применить переписывает
ветку целиком.

## Как устроена кнопка

| Поле | Назначение |
|------|-----------|
| `label` | подпись (всплывает как подсказка) |
| `target` | что запускать |
| `args` | аргументы командной строки (необязательно) |
| `icon` | своя иконка (необязательно) |
| `flags` | доп. флаги (необязательно) |

`target` может быть:

- программой по имени: `notepad.exe`, `calc.exe`;
- полным путём: `C:\Program Files\App\app.exe` (**без кавычек** — аргументы в
  отдельном поле `args`);
- папкой (откроется в Проводнике): `C:\Users\Имя\Downloads`;
- приложением из Store: `shell:AppsFolder\<AUMID>`.

## Флаги кнопки

В редакторе это галочки и выбор блока:

- **admin** — запуск от имени администратора (запрос UAC).
- **left / center / right** — блок выравнивания: левый (по умолчанию), центр,
  правый край.
- **sep** — это не кнопка, а вертикальный разделитель.
- **console** — для `cmd`/`powershell` и других консолей: открыть на **том**
  мониторе, где кнопка (через классический `conhost`). Без этого консоль может
  уехать на другой монитор — её окном владеет Windows Terminal, а не сама команда.

## Блоки (лево / центр / право)

Полосу можно разбить на 2–3 блока. Например: папки слева (без флага), программы
справа (флаг `right`). Часть иконок будет у левого края, часть — у правого.

## Разделители

Чтобы визуально разбить кнопки на группы — вставь разделитель: в редакторе нажми
**«+ Разделитель»** (или у выбранной кнопки поставь галочку «Разделитель»). Это
тонкая вертикальная черта; она тоже слушается края (слева/центр/справа).

## Иконки

По умолчанию иконки берутся из подпапки `ico` **рядом** с `panel.exe` (папка
создаётся сама при запуске). Куда бы ни переместил `.exe` — `ico` ищется рядом
с ним. В поле `icon` пиши просто имя файла (`foo.png`) — оно ищется в этой папке;
или укажи полный путь. Сменить папку — значение `icon_dir` (относительное — рядом
с exe; абсолютное — где угодно).

Форматы: `.ico .png .jpg .bmp .gif .tiff` (**SVG не поддерживается** — заранее
конвертируй SVG в PNG). Любой размер ужимается до 32 px; лучше квадрат 48–256 px,
PNG с прозрачностью.

## Переменные окружения

В `target` / `args` / `icon` можно писать `%USERNAME%`, `%USERPROFILE%`,
`%APPDATA%` и т.п. — подставятся автоматически.

## Разные кнопки на разных мониторах

`monitor_1` — самый левый монитор, `monitor_2` — правее, и т.д. У каждого свой
набор кнопок. На одномониторной машине используется только `monitor_1`.

**В командировке / на одном экране.** Чтобы добраться до кнопок `monitor_2` и
`monitor_3`, когда монитор всего один, открой редактор и либо нажми **⇄** — поменять
набор `monitor_1` местами с нужным, либо выбери сверху другой монитор и перетащи
отдельные кнопки через **ПКМ → Переместить на монитор → Монитор 1**. Настройки при
этом не рушатся — вернёшь тем же способом, когда экраны снова будут на месте.

## Язык интерфейса

По умолчанию **русский**; второй встроенный язык — **английский**. Переключение —
меню **Язык / Language** (выбор сохраняется в реестре; на первом старте язык
определяется по системной локали).

Строки лежат в `lang\<код>.lang` рядом с `panel.exe` (INI, UTF-8). Файлы
`ru.lang` и `en.lang` создаются из встроенных дефолтов при первом запуске (если
их нет) и служат шаблоном; твои правки в них при следующих стартах сохраняются.
Чтобы вернуть исходный шаблон — удали файл и перезапусти. **Свой язык:**

1. Скопируй `lang\en.lang` в `lang\<код>.lang` (например, `de.lang`).
2. Переведи правую часть строк `ключ = значение`; в `lang.name` укажи название
   языка (оно показывается в меню).
3. Перезапусти `panel.exe` — язык появится в меню **Язык / Language**.

Многострочные значения (тексты диалогов) пишутся в одну строку с `\n` на месте
переноса.

## Сборка

Требования: **Visual Studio 2022 Build Tools** + C++ workload (MSVC + Windows
SDK). Без CMake, vcpkg и .NET.

```bat
build.bat
```

Скрипт вызывает `vcvars64`, компилирует `res\panel.rc` (`rc`) и `src\panel.cpp`
(`cl /std:c++17 /O2 /utf-8`), линкует со штатными библиотеками, закрывает
запущенную копию и запускает свежий `build\panel.exe`.

## Переносимость

Работает «из коробки» на другой Win10/11: сам `.exe`, папка `ico`, bare-имена
системных утилит, enum мониторов. **Не переносится** конфиг (он в реестре `HKCU`)
— на новой машине панели стартуют с образцовым набором; перенеси экспортом/импортом
`.reg` или настрой заново в редакторе. Абсолютные пути в `target` и Store-AUMID
тоже могут требовать правки.
